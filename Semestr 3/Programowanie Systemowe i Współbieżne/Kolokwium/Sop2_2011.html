  8. Co wynika z poniższego kodu? 

    #include <fcntl.h>
#include <sys/types.h>
#include <sys/stat.h>

int main(int argc,char **argv)
{
  mkfifo( "plik", 0660 );
  if (fork()) {
    int fd = open( "plik", O_WRONLY );
    if ( fd <0 )
      perror("Blad");
    int n = write( fd, "binfo", 5);
    close(fd);	
  } else {
    int fd = open( "plik", O_RDONLY );
    char buf[10]={0};
    int n = read(fd, buf, 10);

    Proces potomny nie może odczytać zapisaną przez rodzica wartość, ponieważ otwarcie kolejki w trybie odczytu powinno zawsze następować przed jej otwarciem w trybie tylko do zapisu, a tym programie nie jest to zagwarantowane.
    Proces potomny nie odczyta zapisanej wartości, bo rodzic zapisał 5 bajtów, a potomek odczytuje 10 bajtów.

  14. W przypadku kolejek FIFO, otwarcie pliku może spowodować blokowanie:

    Operacje otwarcia pliku w trybie tylko do zapisu są zawsze blokowane, aż inny proces nie otworzy pliku w trybie tylko do odczytu. Operacje otwarcia pliku do odczytu nie są blokowane.
    Jeżeli proces otwiera plik w trybie tylko do odczytu (lub tylko do zapisu), aż do czasu aż inny proces otworzy plik w trybie tzw. komplementarnym (odpowiednio, w trybie tylko do zapisu lub tylko do odczytu).
    Operacje otwarcia pliku w trybie tylko do odczytu są zawsze blokowane, aż inny proces nie otworzy pliku w trybie tylko do zapisu. Operacje otwarcia pliku do zapisu nie są blokowane.
    Nigdy nie powoduje blokowania.

  25. Jak długo istnieją w systemie kolejki FIFO?

    Od chwili ich otwarcia, aż do ich zamknięcia przez ostatni proces korzystający z kolejki.
    Tak długo, jak uruchomiony jest proces, który stworzył kolejkę
    Od chwili ich utworzenia, do ich zamknięcia przez ostatni proces korzystający z kolejki.
    Od chwili ich utworzenia, aż do ich usunięcia.

  27. Po zapisaniu danych do potoku przez funkcję write:

    Dane nigdy nie są usuwane, ale po każdym wywołaniu funkcji read przesuwany jest wskaźnik pozycji pliku dla wszystkich podłączonych procesów. Aby dane odczytać ponownie, należy przesunąć ręcznie wskaźnik pozycji za pomocą funkcji lseek.
    Dane są usuwane po odczytaniu ich przez wszystkie procesy, które są podłączone do potoku.
    Dane są usuwane po ich odczytaniu funkcję read, kolejne wywołania funkcji read nie odczytaj± już tych danych.
    Dane nigdy nie są usuwane, ale po każdym wywołaniu funkcji read przesuwany jest wskaĽnik pozycji pliku dla procesu, który wywołał funkcję read (pozostałe procesy mogą dane odczytać). Aby dane odczytać ponownie, należy przesun±ć ręcznie wskaźnik pozycji za pomocą funkcji lseek.
